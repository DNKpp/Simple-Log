Detailed Description

std::ostream orientated Sink class

This Sink class uses a std::ostream reference for printing each recorded message and offers options to manipulate its behaviour: e.g. filtering and formatting messages. Due to the thread-safe design it is totally fine changing settings during program runtime.

This class offers everything you'll need to print messages into console via std::cout or std::cerr.

void sl::log::BasicSink::log ( const Record &  record)
inlineoverridevirtual

Filters, formats and writes the passed record to the internal stream.

This function prints the passed record to the internal stream object. In forehand, the active filter provides feedback if the record should be passed to the stream or not. If not, the call has no effect to any state. The active formatter will be used to hand-over the necessary information of the Record object to the stream object.

Sets the active filter.

It's the filters job to decide, which Record will be printed (filter returns true) and which will be skipped (filter returns false). Therefore a filter must be an invokable of the following signature:

bool(const Record&)
Remarks
The filters return type doesn't have to be bool, but the returned object must at least be implicitly convertible to bool.

void sl::log::BasicSink::setFormatter ( TFormatter &&  formatter)
inlinenoexcept

Sets the active formatter.

It's the formatters job to:

  • extract the necessary information from the Record
  • pass the extracted infos to the stream object

This design decision is motivated by the fact, that it would be unnecessarily inefficient letting the formatter creating a temporary string object, which would simply get passed to the internal stream. Thus, a custom formatter also has to pass everything to the stream by itself.

A formatter should use the following signature:

void(std::ostream&, const Record&)
Remarks
The formatters return type does not have to be void, but any returned information will be ignored.
